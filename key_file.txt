## Key Directories and Their Functions

- `.github/workflows/`: Contains GitHub Actions configurations for automated CI/CD and deployment.
- `data/`: Stores raw data files used in the analysis.
- `external_data/`: Houses external datasets, including geographical data from Natural Earth.
- `project/`: Contains all Python scripts for backend data analysis, including ECM, price differential, and spatial analysis.
- `public/`: Holds static assets and the HTML template for the React application.
- `results/`: Stores output data from various analyses, accessible to both backend and frontend.
- `scripts/`: Contains utility scripts, such as data preparation for the frontend.
- `src/`: Houses all React components, hooks, utilities, and styles for the frontend application.

## Key Files and Their Functions

### Root Directory
- `config-overrides.js`: Provides configuration overrides for Create React App without ejecting.
- `package.json`: Defines project dependencies, scripts, and other metadata for the Node.js environment.
- `requirements.txt`: Lists all Python dependencies required for the backend analysis scripts.
- `webpack.config.js`: Configures Webpack for bundling the application.
- `map_git_repo.py`: Python script to generate a visual representation of the repository structure.

### Project Directory
- `project/config/config.yaml`: Configuration settings for Python analysis scripts.
- `project/ecm_analysis/ecm_analysis.py`: Implements the Error Correction Model analysis.
- `project/price_differential_analysis/price_differential_model.py`: Analyzes price differentials between markets.
- `project/spatial_analysis/spatial_model.py`: Performs spatial analysis on market data.
- `project/utils/common_utils.py`: Common utility functions used across different analysis scripts.
- `project/utils/data_utils.py`: Utility functions for data manipulation and preprocessing.

### Source Directory (src/)
- `App.js`: The main React component that serves as the entry point for the application's UI.
- `Dashboard.js`: The primary dashboard component that integrates various visualizations and analysis results.
- `index.js`: The entry point for the React application, where the App component is rendered.

#### Components
- `components/common/`: Reusable UI components like Header, Sidebar, and error handling components.
- `components/ecm-analysis/`: Components specific to displaying ECM analysis results.
- `components/interactive_graph/`: Components for interactive data visualization.
- `components/methodology/`: Components for displaying methodology information.
- `components/price-differential-analysis/`: Components for price differential analysis visualization.
- `components/spatial-analysis/`: Components for spatial analysis visualization.

#### Hooks
- `hooks/useData.js`: Custom hook for fetching and managing the main dataset.
- `hooks/useECMData.js`: Hook for fetching and managing ECM analysis data.
- `hooks/usePriceDifferentialData.js`: Hook for managing price differential analysis data.

#### Styles
- `styles/CustomStyles.js`: Custom styled components for consistent UI design.
- `styles/GlobalStyle.js`: Global styles applied to the entire application.
- `styles/theme.js`: Theme configuration for Material-UI components.

#### Utils
- `utils/api.js`: Functions for making API calls to fetch data.
- `utils/dataProcessing.js`: Utility functions for data processing on the frontend.
- `utils/ecmSlice.js`: Redux slice for managing ECM analysis state.
- `utils/priceDiffSlice.js`: Redux slice for managing price differential analysis state.
- `utils/spatialSlice.js`: Redux slice for managing spatial analysis state.

### Scripts
- `scripts/prepareData.js`: Node.js script that prepares and copies necessary data files for the frontend build.
